<objective>
Fix the three critical stability bugs: (1) analysis hanging indefinitely due to sync Anthropic client, (2) race conditions from module-level globals, (3) missing async progress reporting. Convert the entire analysis pipeline from sync to async with per-agent 5-minute timeouts, and replace global session state with per-session state keyed by UUID.

Purpose: The analysis button currently hangs the server if the Claude API stalls. Two concurrent sessions corrupt each other. These must be fixed before any multi-team work.

Output: Fully async analysis pipeline with timeout protection, per-session state isolation, and session-specific progress/error reporting.
</objective>

             try:
                 desk = TradingDesk(db=db, tickers=tickers)
                 await desk.run_trading_session(tickers, on_progress=lambda s, n, t: _update_progress(session_id, s, n, t))

             _sessions[session_id]["error"] = str(e)
